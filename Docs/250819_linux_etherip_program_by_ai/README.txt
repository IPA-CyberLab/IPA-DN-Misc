Linux カーネル 6.8.0 において動作する、EtherIP (RFC3378) の仮想デバイスド
ライバモジュールのソースコード一群を、C 言語で実装せよ。
C ソースのほか、ヘッダファイル、Makefile その他のビルドに必要なファイルを
すべて揃えて、既存の Linux ソースコードツリーへの組み込み方法を提示せよ。
生成するソースコードには、日本語で、各部に詳しいコメントを記載せよ。

仕様:
1. ネットワークインターフェイスとして、eip0, eip1, ... というように "
eip" というシンボル名に見える。
2. 何らかの root が実行可能なコマンドで (echo 等) /proc/sys/net/ システム
システム下の特殊なコントロールファイル (作成用) に指令を飛ばすと、新たな
ネットワークインターフェイス (eip0 等) がシステムに登場する。
また、/proc/sys/net/ 下に、作成された当該新規インターフェイスに対応するコ
ントロール用のファイルが出現される。
これらは、root でのみ書き込みできること。読み込みは、誰でもできること。
3. eip0 等が作成されたなら、当該仮想インターフェイスは、内部的に、次のパ
ラメータを持つこと。
(a) 宛先 IPv4 または IPv6 アドレス 1 つ。
(b) 当該仮想インターフェイスの MAC アドレス (6 バイトの値)。
4. 3 の現在の設定値は、2 の方法で作成されたコントロール用のファイルを通じ
て読み書きできること。ここにおいて、(a) は通常の ASCII テキスト表記でアド
レスを表記することとし、(b) は 001122334455 のような 16 進数の 6 バイトの
文字列 (12 文字 HEX 表記) とすること。
5. 任意の Linux 上のプロトコルスタックが、eip を物理的なインターフェイス
等価とみなして、Ethernet フレームを送付しようとしたら、これを EtherIP で
カプセル化することにより、(a) 宛てに EtherIP パケットとして送付する。この
際には、Linux の既存の TCP/IP スタックを利用して、IP パケットを送付するこ
と。
6. Linux の既存の TCP/IP スタックで EtherIP パケットを受信したときは、そ
のパケットの送信元アドレスが (a) で指定されたアドレスである場合に限り、こ
れを EtherIP でカプセル化解除し、5 と全く逆方向に、あたかも eip デバイス
が物理的な Ethernet フレームを受信したのと同様にふるまうこと。
